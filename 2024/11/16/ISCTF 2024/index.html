小蓝鲨的冒险

parse_str函数缺陷漏洞

@parse_str($b);

这个函数不会检查变量 $b 是否存在，如果通过其他方式传入数据给变量$b ,且当前$b中数据存在，它将会直接覆盖掉。

传入?b=a[0]=s878926199a即可绕过

$a = intval($num, 0);
intval() 函数将 $num 转换为整数，第二个参数 0 表示自动识别进制。如果 0 被指定，则 intval 会自动识别以下情况：

  • 如果 $num 是以 0x 开头，则解释为十六进制。
  • 如果 $num 是以 0 开头，则解释为八进制。
  • 否则，按十进制解析。

2024的八进制是3750,再往前面加个0即可绕过

num=03750

上传文件检查：

  • 如果有文件上传（$_FILES['file']），并且文件大小大于 0，则继续执行。
  • 使用 pathinfo() 获取文件的扩展名和文件名。
  • 构建上传目录（./uploads/）和文件的保存路径。

目录创建：

  • 如果 ./uploads/ 目录不存在，会尝试创建该目录，权限为 755。

文件存储：

  • 文件通过 move_uploaded_file() 被上传到指定目录，文件名通过 md5() 哈希加密处理。
  • 如果文件上传成功，会返回一个成功的消息，并提到上传文件的路径（但是并不直接透露路径）。
  • 如果上传失败，会输出错误消息。

第三关

原理:当我们使用 php://filter时,解码过程中的字符<、?、;、>、 等一共有7个字符不符合base64编码的字符范围会被忽略

合法字符只有A-Za-z0-9\/\=\+,那么我们可以利用编码过滤器进行代码构造

PHP Filter 当中有一种 convert.iconv 的 Filter ，可以用来将数据从字符集 A 转换为字符集 B ，其中这两个字符集可以从 iconv -l 获得，这个字符集比较长，不过也存在一些实际上是其他字符集的别名。
